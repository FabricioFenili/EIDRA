# Role Knowledge Routing Matrix

## Objective
Anchor every institutional role to a directed knowledge set and to the repository surfaces it governs.

| Role layer | Scope | Knowledge basis | Primary surfaces |
|---|---|---|---|
| CEO Council | system arbitration | executive canon, sovereign constitutions, DDIA, macro, asset pricing, statistical learning | 00_governance, 01_constitutions, 12_docs |
| VP | domain mandate | VP canons, constitutions, operating models | 02_vps, 08_src/quant_macro_os/*domain* |
| Directorate | domain subfunction | directorate-specific methods and ledgers | 02_vps/*/directorates |
| Superintendency | workflow ownership | validation, diagnostics, execution playbooks | 02_vps/*/directorates/*/superintendencies |
| Manager | delivery orchestration | interface specs, service contracts, runbooks | 08_src, 12_docs, 11_runs |
| Specialist | implementation | method notes, tests, code scaffolds, domain references | 08_src, 09_tests, 10_notebooks |

## Routing rule
No role acts from generic intuition. Every role acts from:
- constitutional mandate
- routed canon
- mapped repository surface
- declared ledger or service interface
