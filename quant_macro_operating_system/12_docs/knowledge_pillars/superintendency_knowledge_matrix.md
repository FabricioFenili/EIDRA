# Superintendency Knowledge Matrix

Each superintendency must have:
- a declared research or engineering question
- an input contract
- an output contract
- a validation policy
- a retirement or escalation trigger

## Examples
- Leakage Control -> anti-leakage methods, temporal integrity references, feature validation tests
- Regime Inference -> time-series regime models, macro ontology, scenario interfaces
- Kelly Sizing -> portfolio theory, risk budget policy, capital ledger integration
- Drawdown Governance -> ruin analysis, stress engine, escalation ladder
